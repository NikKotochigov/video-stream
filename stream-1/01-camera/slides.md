## Слайд 1 — getUserMedia

**Заголовок:**  
`navigator.mediaDevices.getUserMedia()`

**Буллеты:**
- Точка входа: браузер запрашивает доступ к **камере** и/или **микрофону**
- Возвращает `Promise<MediaStream>` — живой поток, не файл
- Работает только в **secure context** (HTTPS или `localhost`)
- Пользователь должен **явно разрешить** доступ — иначе ошибка

**Код на слайде:**

```js
const stream = await navigator.mediaDevices.getUserMedia({
  video: true,
  audio: false,
});
videoElement.srcObject = stream;
```

**Заметка:**  
Подчеркнуть: `srcObject`, не `src`. Три строки — весь минимальный сценарий.

## Слайд 2 — Цепочка до камеры

**Заголовок:**  
Кто с кем разговаривает

**Схема:**

```
Страница          Браузер              ОС                 Устройство
   │                 │                  │                     │
   │ getUserMedia()  │                  │                     │
   ├────────────────►│  Permissions UI  │                     │
   │                 ├─────────────────►│  драйвер камеры     │
   │                 │                  ├────────────────────►│
   │                 │◄── MediaStream ──┼─────────────────────┤
   │◄── srcObject ───┤                  │                     │
```

**Буллеты:**
- Браузер показывает свой диалог разрешений
- ОС решает, какое физическое устройство использовать
- Если камера занята другим приложением — получим ошибку
- Разрешение можно отозвать в настройках браузера (иконка замка в адресной строке)

**Заметка:**  
Не углубляться в Permissions API — достаточно картинки «страница не лезет в камеру напрямую».

## Слайд 3 — Constraints: пожелания, не приказ

**Заголовок:**  
`MediaStreamConstraints`

**Слева — минимум:**

```js
{ video: true, audio: false }
```

**Справа — с пожеланиями:**

```js
{
  video: {
    width:  { ideal: 1920 },
    height: { ideal: 1080 },
    frameRate: { ideal: 30 },
  },
  audio: false,
}
```

**Буллеты:**
- `ideal` — «хочу, но можно другое»
- `min` / `max` / `exact` — жёстче (exact часто ломается на части устройств)
- Браузер **подбирает** ближайший поддерживаемый режим
- Реальный результат смотрим в `track.getSettings()`

**Заметка:**  
Ключевая мысль: constraints ≠ гарантия. На demo покажем requested vs actual.

## Слайд 4 — Что приходит в ответ

**Заголовок:**  
`MediaStream` и `MediaStreamTrack`

**Схема:**

```
MediaStream
├── video track  (kind: "video")
└── audio track  (kind: "audio")   ← если audio: true
```

**Поля, которые полезно знать:**

| Свойство / метод | Зачем |
|------------------|--------|
| `track.id` | уникальный id трека |
| `track.label` | имя устройства («FaceTime HD Camera») |
| `track.readyState` | `"live"` или `"ended"` |
| `track.getSettings()` | width, height, frameRate, deviceId |
| `track.getConstraints()` | что мы просили |

**`<video>` атрибуты:**

- `autoPlay` — начать воспроизведение
- `playsInline` — не уходить в fullscreen на iOS
- `muted` — без muted autoplay часто блокируют

**Заметка:**  
Audio track на Step 1 можно не запрашивать — фокус на видео. На Step 2 добавим аудио.

## Слайд 5 — Типичные ошибки

**Заголовок:**  
Когда не работает

| Ошибка | Причина | Что делать |
|--------|---------|------------|
| `NotAllowedError` | пользователь нажал «Запретить» | объяснить, как сбросить в настройках сайта |
| `NotFoundError` | нет камеры / отключена | проверить устройство, кабель, VM |
| `NotReadableError` | камера занята | закрыть Zoom, Telegram, другую вкладку |
| `OverconstrainedError` | constraints слишком жёсткие | ослабить `exact`, использовать `ideal` |
| `SecurityError` | не HTTPS и не localhost | деплой только по HTTPS |

**Буллет внизу:**  
В demo — показываем ошибку текстом, не молчим.

**Заметка:**  
На стриме можно намеренно нажать «Block» в другой вкладке (если есть) — или просто рассказать.
