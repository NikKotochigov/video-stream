## Слайд 2 — Почему это не `<video src="...">`

**Заголовок:**  
Видео в браузере — это не просто `<video src>`

**Буллеты:**
- `<video src="file.mp4">` — **готовый файл**, браузер декодирует и показывает
- Камера / микрофон — **живой поток**: устройство → MediaStream → обработка → (сеть) → `<video>`
- Браузер — прослойка между ОС, устройствами и Web API
- Frontend-разработчику полезно видеть **весь путь**, а не только финальный тег

**Схема (текст для слайда):**

```
Файл:     [ mp4 ] ──► decode ──► <video>

Живой:   [ камера ] ──► MediaStream ──► ? ──► WebRTC ──► <video>
                              ↑
                         сегодня разберём
```

**Заметка:**  
Не уходить в детали API — только контраст «файл vs поток». Зритель должен понять: эфир про pipeline, не про HTML-тег.

## Слайд 3 — Что в эфире, а чего нет

**Заголовок:**  
Фокус эфира

**Колонка «Разберём»:**
- `getUserMedia` — доступ к камере и микрофону
- `MediaStream` и треки — включить, выключить, остановить
- Обработка кадров — Canvas и Insertable Streams

**Колонка «Сегодня не трогаем»:**

- ML / сегментация человека

**Буллет внизу:**  
Сначала — **как устроен путь данных**. Эффекты и инфраструктура опираются на эту базу.
